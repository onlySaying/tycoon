# Hamburger Tycoon

## 1. 프로젝트 개요
유니티 엔진으로 제작한 햄버거 타이쿤 게임입니다.
엔진이 제공하는 기능을 사용하는 데서 멈추지 않고,
게임의 핵심 로직을 직접 설계하고 구현하는 것을 목표로 했습니다.

## 2. 개발 동기
에셋과 컴포넌트를 활용한 빠른 구현 과정에서,
내부 동작 원리를 이해하지 못한 채 기능을 조립하는 한계를 느꼈습니다.
이에 자료구조와 알고리즘을 실제 게임 로직에 적용하며,
엔진 동작을 스스로 검증해보고자 본 프로젝트를 진행했습니다.

## 3. 핵심 시스템 설계
### (1) 햄버거 조립 및 레시피 검증
- Stack 자료구조를 사용해 재료를 LIFO 구조로 관리
- 실제 햄버거 조립 순서와 일치하는 구조로,
  플레이어 입력과 레시피를 효율적으로 검증

### (2) 식재료 데이터 관리
- HashMap을 활용해 식재료 데이터를 관리
- 재료 ID 기반 조회를 통해
  재료 추가/삭제 시에도 성능 저하 없이 확장 가능하도록 설계

## 4. 기술적 포인트
- 엔진 기능에 과도하게 의존하지 않고,
  로직을 직접 구현하며 자료구조 선택의 이유를 고민
- 단순히 동작하는 구현이 아닌,
  설계 의도를 설명할 수 있는 구조를 우선

## 5. 개발 범위
- 개인 프로젝트
- 게임 로직, 시스템 설계 및 구현 전반 담당

## 6. 회고
편리한 엔진 환경 속에서도 로직을 직접 설계하며,
게임 플레이 경험을 코드 구조로 풀어내는 과정에 몰입했습니다.
이 경험을 통해 게임 시스템을 논리적으로 분해하고,
선택의 이유를 설명할 수 있는 개발 방식의 중요성을 체감했습니다.
